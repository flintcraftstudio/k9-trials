The two main loops

Most user time is spent in two loops: the competitor loop (browse public → log in → register → wait → see results → maybe challenge) and the admin loop (open event → manage trials → review registrations → assign judges → review challenges). The judge tablet is a sealed third loop.

Empty states are the default

Pre-launch and well past launch, most account/admin lists will be empty. Every list screen in this draft has a real empty-state sketch — not "No data."

Open questions on the map

Withdrawal UX once admin has accepted (A6); concurrent multi-judge trials (D6); admin role on the public site — do they get an "admin" hat or stay invisible? (top bar). Flagged inline with orange chips on the relevant screens.
